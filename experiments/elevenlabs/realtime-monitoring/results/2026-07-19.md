# Result: 2026-07-19

## Outcome

**FAIL — the workspace subscription does not include real-time monitoring.**

This is an entitlement failure, not an API-key permission failure and not a failure of the experiment's WebSocket logic.

## Observed response

The experiment selected the existing `exploration` agent and attempted to enable `conversation_config.conversation.monitoring_enabled` before starting a test conversation. ElevenLabs returned:

```text
HTTP 403
type: authorization_error
code: feature_not_available
status: monitoring_enterprise_only
message: Real-time monitoring is an enterprise-only feature. Please upgrade your subscription.
```

The request ID has intentionally not been preserved because it is operational metadata rather than part of the capability result.

## Side-effect verification

After the rejection, the agent was fetched again. `monitoring_enabled` remained `false`; no agent configuration was changed and no test conversation or telephone call was started.

## Conclusion

The documented monitoring WebSocket and its `contextual_update` command cannot be used by this workspace in its present subscription state. Retrying with the same subscription and credentials cannot change this result. The next valid tests are either:

1. rerun this experiment after ElevenLabs grants enterprise/hackathon monitoring access; or
2. test the non-enterprise architecture in which each carrier conversation reads the shared market state through webhook tools at controlled negotiation checkpoints.
